- git
  - 履歴
    - git log --graph --all
      - 全てのブランチのコミット履歴をグラフ構造で
  - 差分
    - git diff
      - ワークツリーとインデックスの差分
    - git diff --cached
      - インデックスとコミットの差分
  - ブランチ
    - コミットについたラベル
    - ブランチに対してコミットすると最新コミットに追従する
    - HEAD
      - 作業中のコミットを示す特別なラベル
      - 通常、ブランチと一緒に動く
      - Detached HEAD
        - ブランチから外れて特定のコミットを指した状態
        - 過去のコミットの状態を調査している時等
  - タグ
    - コミットについたラベル
    - 最新コミットに追従しない
    - 特定時点の状態を保存したい場合
  - 3Wayマージ
    - オリジナルファイルを考慮して、差分を計算
  - conflict
    - 同じ箇所を同時並行で変更すること
    - 競合の解決
     1. gsで競合ファイル確認
     2. 競合を修正して保存
     3. ga, gc
  - Fast-forwardマージ
    - 変更をしていないmain, devの変更の最新のコミットにmainを持ってくる
    - ログが枝分かれしない
    - git merge --no-ff
      - あえてmergeコミットを作る
  - git cherry-pick
    - 任意のコミットをを取ってくる
    - ここで持ってくるコミットはそのコミットの直前のコミットとの差分
  - リベース
    - ベースを付け替える
    - git rebase そのブランチの親ブランチ
    - あたかも最新の親ブランチからブランチしたかのようになる
  - ログの修正
    - git reflog
    - git rebase -i 起点となるコミット
      - コミット順を入れ替える
      - コミットメッセの変更
      - pick
        - コミットを採用
      - edit
        - コミットを採用するが、修正のため中断
      - drop
        - コミットを採用しない
    - git commit --amend
      - コミットの修正
      - 変更の追加、削除
      - コミットメッセの変更
  - git reset
    - --soft, コミットのリセット
    - --mixed, インデックスのリセット, gaのリセット, git resetのデフォ動作
    - --hard, コミットしていないローカル変更もリセット
  - git stash
    - コミットしたくないけどなくなると困る変更
    - 変更を一時退避
    - git stash save
    - git stash list
  - コミット粒度
   - 単一事項のみを含む、一言でそのコミットを説明できる
   - 現在形で書く

- GitHub
  - リポジトリ内Pull Request
    1. ローカルでブランチ切る
    2. そのブランチにコミット
    3. そのブランチをPush
    4. リポジトリページでCompare & pull request
  - リポジトリ間Pull Request
    - OSS開発
    - リポジトリのPush権限を信頼できる人に限定したいが、開発差分は受け取りたい
